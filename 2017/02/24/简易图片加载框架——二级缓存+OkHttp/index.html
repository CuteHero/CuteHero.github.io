简单图片加载框架——二级缓存+OkHttp

如今图片加载技术已经非常成熟，Glide、Fresco等这些图片加载框架，用法简单，功能强大。但程序员不能止步于使用，这里我们就实现一个简易的图片加载框架，核心内容就是二级缓存，内存缓存和本地缓存。

内存缓存

SoftReference和WeakReference

在以前，实现内存缓存，比较流行的做法是使用SoftReference或WeakReference。但是从Android 2.3 API 9开始，垃圾收集器对软引用和弱引用的回收更积极了，这导致软引用和弱引用无效。而现在大部分应用最低支持4.0，也就是说SoftReference或WeakReference的做法已经不再适用。

LruCache

Android提供了LruCache类，用来实现内存缓存。LruCache将最近引用的对象保留在强引用的LinkedHashMap中，并且在超过其缓存之前将最近最少使用的对象移除出缓存。

本地缓存

外置存储缓存目录

本地缓存可以使用DiskLruCache来实现，这里不这么做，而是将Bitmap缓存到SD卡的应用专属目录/storage/emulated/0/Android/data/{app_package_name}/，这个目录不需要权限，专门用来存放应用相关数据，比如数据库、图片缓存等，当卸载应用时，这个目录也会随之删除。

文件名MD5值

本地缓存最终要以文件的形式保存在文件系统中，以图片Url的MD5值作为文件名。所以还需要一个获取MD5值的工具
